  <action>
    Create `topstepx/references/questioning.md` as the trading-specific questioning guide for the TSX framework. This is NOT a simple rename of GSD's questioning.md -- it requires heavy adaptation.

    **Preserve from GSD questioning.md:**
    - The philosophy section (thinking partner, not interviewer)
    - The how_to_question section (start open, follow energy, challenge vagueness)
    - The anti-patterns section (no checklist walking, no interrogation, etc.)
    - The decision_gate pattern (offer to proceed when ready)

    **Replace/Add for trading domain:**

    Replace the generic `question_types` section with trading-specific question categories. These are NOT a checklist to walk through -- they are domains the questioner should cover naturally:

    1. **Instrument and Market** -- What are they trading?
       - Instrument class: futures (ES, NQ, CL, GC, etc.), specific contracts
       - Market hours awareness: does the strategy need RTH-only or include ETH?
       - Data requirements: what timeframes (1m, 5m, 15m, 1h, daily), how much history

    2. **Strategy Type** -- What's the trading logic?
       - Category: trend-following, mean-reversion, breakout, scalping, statistical arbitrage
       - Signal source: indicator-based, price action, volume, order flow, multi-timeframe
       - Entry/exit rules: what triggers entries, what triggers exits, what are the edge conditions
       - If converting from PineScript: what's the strategy name, does it have known repainting issues

    3. **Execution Model** -- How should signals become orders?
       - Bar-close vs tick-based execution (default: bar-close for safety)
       - Order types: market, limit, stop, trailing stop
       - Bracket orders: take-profit and stop-loss levels (default: always use brackets)
       - Position sizing: fixed size, risk-based, Kelly criterion

    4. **Risk Tolerance** -- What are the guardrails?
       - Max position size (contracts)
       - Max daily loss (dollars)
       - Max drawdown (dollars or percentage)
       - Max concurrent positions
       - These are NON-OPTIONAL. If the user doesn't specify, use conservative defaults.

    5. **Account and Environment** -- Where is this running?
       - Account type: evaluation, funded, live simulation
       - TopStepX API access: do they have it ($29/month), do they have their API key
       - Runtime environment: Node.js, Python, or both
       - Existing code: starting from scratch, adapting existing code, converting PineScript

    Replace the generic `context_checklist` with a trading-specific checklist:
    - [ ] What instrument(s) and timeframe(s)
    - [ ] What strategy type and signal source
    - [ ] Entry/exit rules (concrete enough to implement)
    - [ ] Risk parameters (position size, max loss, brackets)
    - [ ] Account type and runtime environment

    Replace the `using_askuserquestion` examples with trading-specific examples:
    - User says "I want to trade ES": ask about timeframe, strategy type, risk tolerance
    - User says "it should be safe": ask "Safe how?" with options like "Small position sizes", "Always use brackets", "Max daily loss limit", "All of the above"
    - User mentions "my PineScript strategy": ask about repainting awareness, signal confirmation preference

    Use tsx-* naming throughout. Reference safety-patterns.md when discussing risk defaults.

    Target size: 6-8KB (similar to GSD's 5KB but with more domain-specific content).
