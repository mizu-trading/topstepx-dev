**Content transformation functions to implement:**

1. `rewritePaths(content, platform)` — Replace `$HOME/.claude/topstepx/` with the platform-specific prefix. Also handle `@$HOME/.claude/topstepx/` pattern. Platform prefixes:
   - claude: `$HOME/.claude/topstepx/` (no change)
   - opencode: `$HOME/.config/opencode/topstepx/`
   - gemini: `$HOME/.gemini/topstepx/`
   - codex: `$HOME/.codex/topstepx/`

2. `parseFrontmatter(content)` — Extract YAML frontmatter between `---` markers. Return { frontmatter (as object), body (remaining content) }. Use simple line-by-line parsing (no YAML library needed -- fields are flat key-value or simple arrays).

3. `transformCommandForPlatform(filename, content, platform)` — Per-platform command transformation:
   - **Claude**: Path rewrite only, keep .md format, keep filename as-is
   - **OpenCode**: Flatten name (help.md -> tsx-help.md), convert `allowed-tools` array to `tools: {toolname: true}` object format with tool name mapping (Read->read, Write->write, Edit->edit, Bash->bash, Glob->glob, Grep->grep, Task->task, TodoWrite->todo_write, WebFetch->web_fetch, SlashCommand->slash_command), path rewrite
   - **Gemini**: Convert to TOML format (.toml extension). Extract description and body, escape `${VAR}` to `\${VAR}` in prompt content, map tool names to snake_case (Read->read_file, Write->write_file, Edit->replace, Bash->execute_command, Glob->glob, Grep->grep). Output format: `description = "..."\nprompt = """\n...\n"""\n`
   - **Codex**: Convert to skill directory format. Create `skills/tsx-{command-name}/SKILL.md`. Convert `/tsx:command-name` references to `$tsx-command-name`. Replace `$ARGUMENTS` with `{{TSX_ARGS}}`. Prepend adapter header explaining tool translations. Path rewrite.

4. `transformAgentForPlatform(filename, content, platform)` — Per-platform agent transformation:
   - **Claude**: Path rewrite only
   - **OpenCode**: Convert tools list to object format (same tool mapping as commands), path rewrite
   - **Gemini**: Convert tools to snake_case names, remove `color` field, add `name` field from filename, path rewrite
   - **Codex**: Convert to skill format in `skills/tsx-{agent-name}/SKILL.md`, path rewrite

5. `copyWithTransform(srcDir, destDir, transformFn)` — Recursively copy a directory, applying transformFn to each file's content. Use `path.posix.join()` or `.replace(/\\/g, '/')` for content paths to avoid Windows backslash issues.

**Install function update:**
Replace the single `copyDirSync(skillSrc, dest)` call with 4 copy operations per platform:
1. Copy commands with `transformCommandForPlatform`
2. Copy agents (from `topstepx/agents/`) with `transformAgentForPlatform`

4. Copy skills (from `skills/`) directly (no transformation needed -- skill format is universal)

- For Codex agent format, use the simpler skills/SKILL.md approach (not config.toml merging) per the research recommendation for v1
- All path rewriting must use forward slashes regardless of OS

The `uninstall(platforms, scope)` function must:

   - Any `gsd-*` files
